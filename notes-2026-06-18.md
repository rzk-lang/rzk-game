# Notes

Ideas in no particular order:

- Level's Prelude should probably be hidden by default
- Prelude should be formatted (see sHoTT Code Style)
- User's code should probably be auto-formatted (rzk has the capability) on edit (with a checkbox to stop doing that)
- Prelude should be highlighted as well
- moves should replace ? with a term in parentheses (...), to avoid syntax errors
- move buttons render weirdly (move name and term form a contiguous string, e.g. "give second (first (is-segal-A ? ? ? ? ?))"), it would be better to have a two-part button (e.g. [give| second (first (is-segal-A ? ? ? ? ?)) ] ), with color-coded left part (move kind) and syntax-highlighted right part (filler term)
- "Inventory" on the level page seems obsolete/stale (remove?)
- I think it might be good to organize levels into "modules" or "sections" and encourage a mix of BOPPPS-style lesson and puzzle game structure for such a collection of levels
